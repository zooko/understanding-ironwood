Ironwood, in one picture

Each bar is all the ZEC that can change hands. The dashed line is the 16 million that are supposed to exist.

A bug in the old Orchard pool means counterfeit ZEC might exist inside it. The pool is private, so nobody can count what’s in there — the honest answer to “how much ZEC can circulate?” is 16M + ?. The moment Ironwood activates, the answer becomes ≤ 16M, and anyone running a node can verify it. Nobody has to trust anyone’s opinion about whether the bug was ever exploited.

How can sealing one pool fix the whole supply?

Ironwood adds one rule: the old pool is sealed. The only move left is out, through the turnstile — a public tally that is already part of today’s protocol. It admits at most as much out as went in: 4.5M. So the most that can ever circulate is 11.5M + 4.5M = 16M, no matter how much counterfeit might be hiding inside. That’s the whole trick.

The fine print: what if counterfeit really is in there?

The turnstile counts amounts, not legitimacy. If counterfeit exits, an equal amount of honest ZEC inside can never leave — it is effectively destroyed. This risk is not created by Ironwood: it has existed quietly for as long as the bug has. What Ironwood adds is the guarantee above, which holds either way: even in the worst case, the bar never passes 16M.

The practical advice falls out directly: migrate out of the old pool early.
